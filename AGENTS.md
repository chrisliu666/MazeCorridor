# AGENTS.md

本文件为 Codex 和其他编码代理在本仓库中工作的说明。后续协作、实现和优化默认使用中文记录。

## 当前项目状态

本仓库用于开发一个浏览器端迷宫解密网页小游戏。

项目中文名：迷宫回廊

项目英文名：`MazeCorridor`

## 当前实现文件

- `index.html`：页面结构、迷宫容器、控制区、看板娘、对局记录。
- `styles.css`：清新古典风格、萌化二次元视觉、迷宫和按钮样式。
- `app.js`：迷宫生成、种子码、鼠标选方向、路径追踪、死路判错、记录等核心逻辑。
- `assets/mascot-reference.svg`：基于用户提供图片风格制作的 Ada 头像资源。
- `dev-server.js`：本地静态开发服务器，默认端口 `5188`。

## 当前产品需求

实现一个迷宫解密网页小游戏，要求：

- 迷宫采用确定性种子算法生成完美迷宫（DFS 递归回溯）。
- 迷宫为矩形，单入口单出口，无环。
- 迷宫难度通过尺寸分级。
- UI 风格清新、古典、耐看。
- 在清新古典的基础上加入适度萌化和二次元交互特效。
- 看板娘名字为 `Ada`，头像使用用户提供图片风格的图片资源，用于反馈、陪伴和氛围营造，但不能抢占迷宫主体。
- 重点打磨鼠标交互。
- 当前优先适配电脑端鼠标操作，移动端不是第一优先级。
- 使用清晰的 `(2h+1)×(2w+1)` 迷宫渲染网格。

## 当前已确认构想

- 迷宫方式：确定性种子算法生成完美迷宫（DFS 递归回溯）。
- 难度设置：通过迷宫尺寸分级。
- 二次元表现：需要看板娘 Ada。
- 数据保存：对局记录保存在当前浏览器本地。
- 设备优先级：以电脑端鼠标操作为主。

## 当前 UI 控件要求

界面需要提供以下按钮：

- 新迷宫：生成一道新的迷宫。
- 重置：回到迷宫入口重新开始。
- 再来一局：在失败后出现，用于快速开始一局新游戏。

按钮需要清晰、易点击，视觉上与"清新、古典、萌化、二次元"的方向一致。

## 当前页面操作说明

- 页面需要提供清晰、简洁的操作说明区域。
- 操作说明应覆盖点击通道选方向、死路判错、出口胜利、重置、种子码等核心玩法。
- 操作说明应放在不干扰迷宫的位置，优先作为右侧控制面板中的紧凑说明卡。
- 操作说明文案要短，便于玩家快速扫读。

## 当前特效要求

- 路径逐格高亮追踪、岔路口可选通道脉冲发光、死路红色闪烁，可以加入轻量动效。
- 动效风格偏萌化、二次元，例如柔和高亮过渡、脉冲呼吸效果。
- 特效必须克制，不能遮挡迷宫通道，不能干扰玩家连续操作。
- 特效需要保持性能稳定，避免长时间动画或大量粒子。
- 看板娘元素应服务于操作反馈和氛围，不应抢占迷宫主体。

## 当前交互规则

- 鼠标点击通道：在岔路口，点击相邻的可通行通道格，选择要走的方向。
- 路径自动追踪：选定方向后，系统沿走廊自动前进到下一个岔路口、死路或出口。
- 路径逐格高亮动画：追踪过程中，通道逐格以粉红高亮显示。
- 各通道格不需要额外交互，仅在岔路口有可点击选项。

## 当前失败机制

- 每局共有 `3` 次错误机会。
- 走到死路时，错误次数 `-1`。
- 死路触发时，路径高亮为红色闪烁效果，玩家自动回到岔路口。
- 错误次数扣完后，本局游戏失败。
- 游戏失败后，需要显示"再来一局"按钮。

## 岔路口机制

- 岔路口指有 3 条或以上可通行方向的格子（入口可以有 2 条可通行方向也算岔路口）。
- 在岔路口，所有可通行的相邻通道格会发出脉冲发光效果。
- 玩家点击任意可通行通道格即选定方向。
- 系统沿选定方向自动追踪至下一个岔路口、死路或出口。

## 当前结算与对局记录

- 游戏需要记录每局结果。
- 成功通关时，记录成功状态、通关用时和种子码。
- 失败时，记录失败状态、失败用时和种子码。
- 对局记录需要在界面中可查看。
- 对局记录保存在当前浏览器本地。
- 对局记录中的种子码应方便玩家复制或重新输入，以便复现该迷宫。
- 用时从本局开始后计时，到成功通关或失败时停止。
- 使用"再来一局"开始新游戏时，应生成并记录新的对局。

## 当前迷宫生成

- 使用 DFS 递归回溯算法生成完美迷宫。
- 每个迷宫有且仅有一条路径连接任意两个格子。
- 单入口（左上角）和单出口（右下角）。
- 入口位于 `(0, 0)`，出口位于 `(h-1, w-1)`。
- 迷宫完全由种子码确定性生成，同一种子码生成完全一致的迷宫。
- 迷宫无需联网加载。

## 当前难度与迷宫尺寸

- 简单：`11×11` 迷宫格（渲染 `23×23`）
- 普通：`17×17` 迷宫格（渲染 `35×35`）
- 困难：`25×25` 迷宫格（渲染 `51×51`）
- 专家：`35×35` 迷宫格（渲染 `71×71`）

## 当前种子码机制

- 每局随机生成一个种子码。
- 种子码格式：`难度-日期-随机串`，例如 `MEDIUM-20260522-A3F8E2`。
- 种子码需要在界面中展示，方便玩家记录或分享。
- 用户可以手动输入种子码，重复进入同一迷宫。
- 同一种子码应稳定复现相同迷宫。
- 新迷宫按钮默认生成新的随机种子码；如果用户输入种子码，则按输入种子码加载对应迷宫。

## 当前开发命令

当前是无需构建工具的静态网页项目。

推荐本地运行命令：

```bash
node dev-server.js 5188
```

访问 `http://127.0.0.1:5188/`。

检查命令：

```bash
node --check app.js
```
