# 迷宫回廊

这是一个浏览器端迷宫解密网页小游戏项目。

英文名：`MazeCorridor`

游戏网址：https://chrisliu666.github.io/MazeCorridor/

## 当前实现文件

- `index.html`：页面结构、迷宫容器、控制区、看板娘、对局记录。
- `styles.css`：清新古典风格、萌化二次元视觉、迷宫和按钮样式。
- `app.js`：迷宫生成、种子码、鼠标选方向、路径追踪、死路判错、记录等核心逻辑。
- `assets/mascot-reference.svg`：基于用户提供图片风格制作的 Ada 头像资源。
- `dev-server.js`：本地静态开发服务器，默认端口 `5188`。

## 产品目标

构建一个轻量、流畅、适合鼠标操作的矩形迷宫解密网页小游戏。整体体验以交互为重点，视觉风格要求清新、古典、耐看。

## 迷宫生成

- 使用 DFS 递归回溯算法生成迷宫，并保证入口处至少有两个可选方向。
- 迷宫为矩形，单入口和单出口。
- 入口与出口由种子确定性选择在迷宫边界上。
- 入口和出口会在渲染网格边界打开通道，并在迷宫中标记为 `S` / `E`。
- 迷宫完全由种子码确定性生成，同一种子码生成完全一致的迷宫。

## 难度与迷宫尺寸

- 简单：`11×11` 迷宫格（渲染 `23×23`）
- 普通：`17×17` 迷宫格（渲染 `35×35`）
- 困难：`25×25` 迷宫格（渲染 `51×51`）
- 专家：`35×35` 迷宫格（渲染 `71×71`）

## 核心交互

- **岔路口**：指有 3 条或以上可通行方向的格子（入口处有 2 条可通行方向也算岔路口）。
- **点击通道**：在岔路口，点击相邻的可通行通道格，选择要走的方向。
- **路径自动追踪**：选定方向后，系统沿走廊自动前进到下一个岔路口、死路或出口。
- **路径逐格高亮动画**：追踪过程中，通道逐格以粉红高亮显示。

## 失败机制

- 每局共有 `3` 次错误机会。
- 走到死路时，错误次数 `-1`。
- 死路触发时，路径高亮为红色闪烁效果，玩家自动回到岔路口。
- 错误次数扣完后，本局游戏失败。
- 游戏失败后，显示"再来一局"按钮。

## 种子码机制

- 每局随机生成一个种子码。
- 种子码格式：`难度-日期-随机串`，例如 `MEDIUM-20260522-A3F8E2`。
- 种子码需要在界面中展示，方便玩家记录或分享。
- 用户可以手动输入种子码，重复进入同一迷宫。
- 同一种子码应稳定复现相同迷宫。
- 新迷宫按钮默认生成新的随机种子码；如果用户输入种子码，则按输入种子码加载对应迷宫。

## 结算与对局记录

- 游戏需要记录每局结果。
- 成功通关时，记录成功状态、通关用时和种子码。
- 失败时，记录失败状态、失败用时和种子码。
- 对局记录需要在界面中可查看。
- 对局记录保存在当前浏览器本地。
- 对局记录中的种子码应方便玩家复制或重新输入，以便复现该迷宫。
- 用时从本局开始后计时，到成功通关或失败时停止。
- 使用"再来一局"开始新游戏时，应生成并记录新的对局。

## UI 方向

- 风格清新、古典。
- 迷宫布局安静、清晰、易读。
- 使用 `(2h+1)×(2w+1)` 渲染网格的矩形迷宫。
- 墙壁深褐底色，通道浅色底色，界限分明。
- 迷宫周围避免过度装饰，重点突出路径选择和路径追踪动画。
- 使用克制的颜色、细腻的边框和有质感的字体。
- 在清新古典的基础上加入适度萌化和二次元氛围，但不能影响迷宫可读性和操作效率。
- 看板娘名字为 `Ada`，头像使用用户提供图片风格的图片资源，用于反馈、陪伴和氛围营造，但不能抢占迷宫主体。

## UI 控件

界面需要提供以下按钮：

- 新迷宫：生成一道新的迷宫。
- 重置：回到迷宫入口重新开始，并重启本局计时。
- 显示轨迹 / 隐藏轨迹：切换已走路径显示。
- 你自己走：自动寻路到出口，用于演示或快速验证迷宫。
- 再来一局：在失败后出现，用于快速开始一局新游戏。

## 操作说明

- 页面在右侧控制面板中提供清晰、简洁的操作说明区域。
- 操作说明覆盖点击通道选方向、死路判错、出口胜利、重置、种子码等核心玩法。
- 重置会清空当前路径、错误次数和计时，从入口重新开始同一迷宫。
- 操作说明文案简短，便于玩家快速扫读。

## 特效

- 路径逐格高亮追踪、岔路口可选通道脉冲发光、死路红色闪烁，可以加入轻量动效。
- 动效应偏萌化、二次元，例如柔和高亮过渡、脉冲呼吸效果。
- 特效必须克制，不能遮挡迷宫通道，不能干扰玩家连续操作。
- 特效需要保持性能稳定，避免长时间动画或大量粒子导致页面卡顿。
- 看板娘元素服务于反馈和氛围，不应抢占迷宫主体。

## 开发

当前是无需构建工具的静态网页项目。

推荐本地运行命令：

```bash
node dev-server.js 5188
```

然后访问 `http://127.0.0.1:5188/`。

当前检查命令：

```bash
node --check app.js
```
